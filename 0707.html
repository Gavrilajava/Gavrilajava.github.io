The use of libraries to control the state of such as Redux is a common practice when creating React applications. While for large applications this is really justified and convenient, using a third-party library to transfer state between two components may not seem appropriate. Fortunately, React has a built-in alternative called useReducer(). useReducer() is a method from the React Hooks API, similar to useState() but giving more control over state management. It takes the reducer function and the initial state as arguments, and returns the state and the dispatch method: Reducer (so-called because of the type of function which is passed to the array method — Array.prototype.reduce(reducer, initialValue)) is a template taken from Redux. If you are not familiar with Redux, in short, the reducer is a function that takes the previous state and action as arguments and returns the next state. The actions are a piece of information that describes what will happen. And based on this information, the reducer defines how the state should change. Actions are passed through dispatch(action) method. Most of the time you’ll be fine with just usingState(), which is based on useReducer(). But there are cases where it is still preferable to useReducer(). The following state depends on the previous It is better to use this method when the next state depends on the previous one.
